Diferentes tecnologías

URL:

Uniform Resource Locator (URL) es un string de texto que especifica en donde se encuentra un recurso como una página web, una imagen o un video en el internet.

MIME:

Multipurpose Internet Mail Extensions (MIME) es un estándar que indica la naturaleza y formato de diferentes tipos de archivos como archivos de texto, gráficas, o audio para que puedan ser enviadas a través de la internet y puedan ser vistos o reproducidos por un navegador o una aplicación de email.

HTML5:

HTML es el lenguaje estándar de páginas web. HTML5 es le versión estable más nueva del lenguaje HTML, con este se puede estructurar desde un documento hasta una aplicación de desarrollo web. HTML5 incluye nuevos elementos los cuales permiten un mejor manejo de memoria, contenido multimedia y acceso a hardware.

JavaScript:

JavaScript es un lenguaje de programación conocido principalmente como una excelente herramienta para el desarrollo de páginas web. Sin embargo, también se utiliza en otros tipos de ambientes como Node.js, Apache CouchDB y Adobe Acrobat. JavaScript permite que el contenido de una página web sea actualizado sin necesidar de recargar la página.

php:

PHP es un lenguaje de scripting, es una podera herramienta para la creación de páginas web las cuales necesitar ser dinámicas e interactivas.

jsp:

JavaServer Pages (JSP) es una tecnología para desarrollar páginas web las cuales pueden admitir contenido dinámico.

Ajax:

Ajax es una manera de acercarse al desarrollo de páginas web. Este sirve con tecnologías como HTML, CSS y JavaScript. Cuando estas tecnologías se combinan con Ajax, las aplicaciones web pueden hacer actualizaciones rápidas a la interfaz sin necesidad de vovler a cargar la página web.

CÓDIGOS DE ERROR


Existen muchos códigos de error entre ellos:
404 Not Found: La página solicitadano pudo ser encontrada pero es posible que este disponible de nuevo en el futuro.

403 Forbidden: El acceso a el recurso solicitado está prohibido.

500 Internal Server Error: Un error genérico, este mensaje de error se da cuando no hay mensajes de error que especifiquen de mejor manera el error que esta ocurriendo.

503 Service Unavailable: El servidor no está disponible, se encuentra sobrecargado o no esta en línea.

504 Gateway Timeout: El servidor actúa como una puerta, camino o proxy y no recibió respuesta del servidor dentro del tiempo establecido.

**Preguntas a responder**

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red de redes, utiliza protocolos TCP/IP los cuales permiten la conexión y el intercambio de datos entre dos o más anfitriones (hosts).
World Wide Web (WWW) es básicamente una colección enorme de información la cual es accesada a través de la internet.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: El método GET es utilizado para pedir datos de una fuente específica.

HEAD: El método HEAD es utilizado para saber que recibirá un método GET sin recibir como respuesta el cuerpo regresado por el método GET.

POST: El método POST es utilizado para enviar datos a un servidor. Se puede crear o actualizar un recurso dentro de ese servidor.

PUT: El método PUT es utilizado para enviar datos a un servidor, se puede crear o actualizar un recurso.

PATCH: El método PATCH se utiliza para aplicar modificaciones parciales a un recurso.

DELETE: El método DELETE borra un recurso específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se utiliza el método POST ya que la información se encuentra escondida en el cuerpo de la petición al servidor.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET ya que se requiere obtener los datos de una fuente específica.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No ocurrió ningun error. La respuesta 200 significa que todo esta bien (OK), significa que la respuesta a la petición fue exitosa.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentóacceder al sitio y se encontró con un error 404? ¿Por qué?

No, ya que es posible que la página no este disponible en ese momento o que el enlace utilizado es un enlace roto.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

En este caso si es responsabilidad del desarrollador ya que es un error interno del servidor.

HTML5 introduce tags nuevos además de desaprobar de tags utilizados en HTML4.
HTML5 es más simple que HTML4.
HTML5 es mejor en el manejo de errores.\

¿Qué componentes de estructura y estilo tiene una tabla?

< table > define una tabla en HTML.
< tr > define las filas de la tabla.
< th > define el header de la tabla.
< td > define los datos por celda.
< th > los elementos dentro de este tag están en negritas y centradas.
< td > los elementos dentro de este tag están alineados a la izquierda.
