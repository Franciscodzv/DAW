¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

Es importante utilizar estas funciones para tener un mejor control sobre la informacion que se utiliza, nos da extra seguridad además de que nos da código más limpio.

¿Cuál es la diferencia entre una variable de sesión y una cookie?

La variable de sesión guarda información la cual se puede utilizar en diferentes páginas y no se guarda en la computadora del usuario, a diferencia de una cookie la cual si se guarda en la computadora del usuario.

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Facebook asigna un código único a la foto y lo guarda en una variable de sesión. De esta manera el usuario puede subir la misma foto dos veces y facebook simplemente asigna dos códigos distintos a las fotos.

¿Qué es CSRF y cómo puede prevenirse?

Un CSRF es un ataque que utiliza a una víctima a enviar una solicitud maliciosa a un sitio web vulnerable. Esto puede ser realizar una compra, cambiar de contraseña, etc. Este tipo de ataque se puede prevenir al implementar un token anti-CSRF. Un token anti-CSRF es un string random el cual solo lo conoce el navegador del usuario y la aplicación web, este string se guarda en una variable de sesión, cuando el usuario manda alguna solicitud la aplicación web compara ambos strings y si son iguales acepta la solicitud, de otra manera la rechaza.

Otra manera es verificando las cookies del usuario. Como cada cookie es unica para cada usuario, la aplicación web checa si la cookie del usuario proviene del mismo sitio y así comprueba que el usario está utilizando su sesión y que la solicitud no proviene de otro sitio.
